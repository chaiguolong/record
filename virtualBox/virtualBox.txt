1. virtualbox挂载目录失败mount: unknown filesystem type ‘vboxsf’
	解决方法安装VirtualBox增强工具
	0. 先安装必要的包:执行下面的命令:
		> 1. yum install -y gcc gcc-devel gcc-c++ gcc-c++-devel make kernel kernel-devel
		> 2. 安装完重启虚拟机
			shutdown -h now
	1. 打开mac os => /Applications/VirtualBox.app/Contents/MacOS
	2. 找到VBoxGuestAdditions.iso 加载到虚拟机
		> 1. 点击设置-->存储-->控制器IDE
		> 2. 点击左边第一个小按钮(添加虚拟光驱)
		> 3. 选择VBoxGuestAdditions.iso
	3. 在虚拟机当中执行(先cd 到/media,然后mkdir cdrom,最后给777权限)
		sudo mount /dev/sr0 /media/cdrom
	4. cd /media/cdrom/ && sh ./VBoxLinuxAdditions.run
	5. reboot虚拟机即可.
2. 在Virtualbox中与主机共享文件夹
	1)完成👆(上面)的步骤是关键.
	2).点击虚拟机 设置-->选择共享文件夹
	3).设置共享文件夹
		> 1.选择路径
		> 2.填写自定义的共享名称(后面需要与挂载路径相对于.
			这很重要!!!,我这里是名称是test),这里填的是名称,
			不是路径,路径在上一步选择.
		> 3.设置自动挂载/固定分配.
	4).挂载共享文件夹
		切换到root用户输入挂载命令:
		sudo mount -t vboxsf test /mnt/test
		##上面命令中test是步骤2中的名称;后面的/mnt/test是挂载目录,
		##需要先创建目录,赋予权限777,才能挂载.

2.在虚拟机中桥接模式和NAT模式的区别.

1)概要:
		桥接	通过使用物理机网卡 具有单独ip(就想局域网中的独立主机,与宿主机同一网段)
		NAT		把物理机为路由器进行上网(将主机当做路由器,然后在此路由器下虚拟出的主机,是宿主机的下一级,不是同一网段)

2)简介:

	桥接:
		虚拟机操作系统和实体机操作系统同时使用真实网卡工作,
	虚拟出来的操作系统就像是局域网中的一独立的主机,它可以通
	过路由器网线访问网内任何一台机器.不过虚拟机需要占用你同
	一个网段的一个IP地址,当且仅当虚拟机和实体机在同一个网段,
	两者才可以进行通信.如:
		虚拟机IP:192.168.1.101
		实体机IP:192.168.1.100

	数据流:
		虚拟机访问真实机:	虚拟机系统->真实网卡->路由器->真实网卡->实体机系统
					(不难看出同一个数据经过网卡在路由器上面绕了一圈又回到了网卡)
		虚拟机访问外网:		虚拟机系统->真实网卡->路由器->外网

	优点:
		相当于是独立的主机,与宿主机无区别.
		如果你想利用VirtulBox在局域网内新建一个虚拟服务器,为局域网用户提供网络服务,就应该选择桥接模式

	缺点(主机必须要连接网络):
		1).桥接需要单独配置网络,相对复杂(将桥接的虚拟主机当做独立主机,与宿主机要在同一网段).
		2).桥接必须经过真实网卡和路由器,因此当你的电脑没有链接有线网或者无线网的话,
	你的实体机是无法和虚拟机进行交互的,即便看起来两者很近也不行

	

	NAT:
		NAT模式实际是虚拟了一个网卡出来，虚拟机直接使用链接这个虚拟网卡,每次访问和交互通过这个虚拟网卡交换数据

	数据流:
		虚拟机访问宿主机:	虚拟机系统->虚拟网卡->实体机系统（可以发现是不经过真实网卡的流程简单很多);
		虚拟机访问外网:		虚拟机系统->虚拟网卡->实体机系统->真实网卡->路由器->外网

	优点:
		虚拟系统接入互联网非常简单，你不需要进行任何其他的配置，只需要宿主机器能访问互联网即可.

	缺点:
		NAT模式下,虚拟机仅仅可以同路由器下网段中的一台真实机通讯,而这台真实机就是安装虚拟机的这台电脑.

3).配置:

	桥接:
		1).配置网卡,选择桥接网卡模式,并且指定桥接的具体网卡.
		(在虚拟机中选择设置->网络->网卡1->桥接网卡->en0:Wi-Fi(AirPort))
		
		2).查看宿主机的网卡信息,可以确认我们需要的网卡地址
		(192.168.0.1),子网掩码(255.255.255.0),ip(192.168.0.104)

		3).修改网卡配置文件,实现静态IP连通内外网.
			3.1).进入/etc/sysconfig/network-scripts/目录,编辑ifcfg-enp0s3文件(名称可能不一样)
			3.2).参数说明:

				BOOTPROTO=static(静态IP)
				IPADDR=192.168.0.201(IP地址,此IP地址要配置和宿主机同一网段)
				GATEWAY=192.168.0.1(要和宿主机同一网关)

		4). 环境查看:
		外网可以连接()
			4.1).宿主机可以连接外网
				(例如ping www.baidu.com)
				如果宿主机不能连接外网,那么虚拟机和宿主机之
				间就不能相互访问,因为它们之间访问需要通过网络

			2.1).虚拟机可以访问物理机和外网
				ping 宿主机ip(在虚拟机)
				ping www.baidu.com

			2.2).物理机可以访问虚拟机
				ping 虚拟机ip(在宿主机)


3.NAT模式下通过端口转发实现宿主机访问虚拟机,实现ssh连接.
	1).在虚拟机设置中选择网络,然后选择网络地址转换NAT.
	2).点击高级,点击端口转发.
	3).新建一条规则(其中主机端口自由选择,子系统IP为虚拟机IP,最好设置为静态IP,方便连接),例如:
		名称	协议	主机IP		主机端口	子系统IP	子系统端口
		Rule1	TCP		127.0.0.1	8888		10.0.2.21	22
	4).点击OK,应用即可.
	5).宿主机通过ssh连接虚拟机.
		ssh -p 8888 root@127.0.0.1
	
			

		
4.本地浏览器访问虚拟机的hugo服务
	1.方法一:
		服务器用hugo server不带参数启动服务
		用ssh登录,加一下参数-L 1313:localhost:1313
		然后本地浏览器直接访问http://localhost:1313
		ssh -L 1313:localhost:1313 dream@192.168.0.105
	2.方法二:
		在服务端bolog启动服务时候运行下列命令
		hugo server --bind 192.168.0.105 -b 192.168.0.105
